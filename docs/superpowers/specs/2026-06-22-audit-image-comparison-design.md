# AI审计分析平台 — 骗补行为图像检测改造设计

## 概述

改造现有 AI 智能审计分析平台的中央面板功能，聚焦于**报废农业用具骗补行为检测**。

### 两个审计场景

1. **图片查重**：不同人上传相同农具照片 → 骗报废补贴嫌疑
2. **条码不一致**：条形码图案相同但数字不同 → 骗报废补贴嫌疑

### 改造范围

仅改动中央面板（上传 → 分析 → 报告）和右侧图片库面板。左侧面板和顶部导航保持不变。

### 约束条件

- 纯前端项目，无后端
- GLM Vision API（OpenAI 兼容格式）实现图片对比
- API Key 由用户在页面配置，存入 localStorage
- 图片对比模式：1 对 N（每张上传图 vs 图片库所有基准图）
- 自动判断图片类型（实物照片 / 条码）

---

## 架构

### 整体流程

```
上传图片（多张）
      ↓
分析配置 & API Key 检查
      ↓
阶段1：逐张识别图片类型（实物照片 vs 条码）
      ↓
阶段2：按类型分组，批量对比图片库
      ↓
阶段3：汇总结果，生成审计报告
```

### 模块划分

| 模块 | 职责 | 涉及文件 |
|------|------|---------|
| 上传模块 | 文件拖拽/选择、缩略图预览、格式校验 | audit_agent.html（已有）、js/app.js |
| 图片库模块 | 基准图片库管理（增/删/浏览） | audit_agent.html（改造右侧面板）、js/app.js |
| API 模块 | GLM API 调用封装、Key 管理 | js/api.js |
| 分析引擎 | 类型识别 → 预筛选 → 深度对比 → 报告生成 | js/analysis-engine.js |
| 配置模块 | API Key 设置、模型选择、参数调整 | 中央面板配置区 |
| 报告模块 | 结果展示、风险分布、导出 | js/app.js（已有报告区骨架） |

## 右侧面板重新设计

- 网格展示所有基准图缩略图
- 顶部显示图片总数
- 每张图：可点击放大预览、可删除

- 支持滚动查看

## 阶段1：类型识别

### 输入

上传的每张图片（base64）

### 处理

- 每张图单独发送到 GLM API
- 同时提取 EXIF 元数据（拍摄时间、设备型号、GPS）
- 本地计算感知哈希（pHash）供阶段2预筛选

### API Prompt 设计

```
请判断这张图片属于以下哪种类型：
A. 实物照片 — 拍摄的农业用具、机械设备等实物
B. 条码/二维码 — 包含条形码或二维码的图片

请用JSON格式回复：
{"type": "A或B", "description": "简短描述图片内容", "quality": "清晰/模糊/可疑"}
```

### 输出

每张图标注：类型、内容描述、质量评估、EXIF 摘要、pHash 值

## 阶段2：对比分析

### 预筛选（减少 API 调用量）

- 用上传图的 pHash 与库中所有图的 pHash 计算汉明距离
- 距离 < 阈值（如距离 ≤ 10）的进入深度对比
- 预筛选为本地计算，不消耗 API 调用
- 过滤掉约 90% 的无关图片

### 实物照片深度对比

将 1 张上传图 + N 张候选库图（预筛选后的）编为一组，发送给 GLM：

```
图A是待审的上传图片（报废农具），图B~图E是基准库中的图片。
请逐一对比图A与图B~图E：
1. 是否拍摄于同一件物品？（考虑角度、光照差异）
2. 相似度评分（0-100）
3. 相似度>80%的记录为可疑
4. 若图A与同一批次的其他上传图也相似，一并标注

仅返回JSON数组：
[{"match_image": "库图名", "similarity": 分数, "is_suspicious": true/false, "reason": "简短说明"}]
```

### 条码深度对比

```
这是待审图片中的条码。请先提取：
1. 条码类型（EAN-13/Code-128/QR等）
2. 条码下方/周围的数字文本
3. 条码图案对应的编码内容

库中有以下基准条码图片...请逐一比对：
- 条码图案是否与库中某图相同或高度相似？
- 如果图案相同但数字文本不同 → 标记为骗补可疑
- 如果图案相同且数字也相同 → 标记为重复上传

仅返回JSON数组。
```

### 批次内自查

额外检查：同一批次上传的图片之间是否有相互重复的情况。

### 并行策略

- 每张上传图的对比任务互不依赖，可并行发送 API 请求
- 每批 API 调用完成后立即更新 UI 状态

### 输出

每张上传图的对比结果列表，含匹配项、相似度、是否可疑、原因描述。

---

## 阶段3：报告生成

### 汇总数据

将所有对比结果（可疑匹配列表、无匹配项、条码异常项）汇总。

### API Prompt

```
以下是本次审计的图片比对汇总：
[插入所有对比结果的JSON摘要]

请生成一份审计分析报告，包含：
1. 总体评估（共上传X张，发现可疑Y项）
2. 可疑项详情列表（每项包含：图片名、匹配来源、相似度/异常描述、风险等级）
3. 统计概览（按类型/风险等级分布）
4. 整改建议（针对发现的骗补嫌疑提出审计处理建议）

风险等级定义：
- 高风险：相似度>90%，基本确认同一物品
- 中风险：相似度70-90%，需要人工复核
- 低风险：有疑点但证据不够充分
```

### 输出

Markdown 格式的审计报告，嵌入生成的对比图，在页面中央面板展示。

## 配置模块

在中央面板上传区上方或分析开始前，提供配置入口：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| API Key | 智谱 API 密钥 | 空（必填） |
| API 地址 | 可自定义端点 | https://open.bigmodel.cn/api/paas/v4 |
| 模型选择 | 视觉模型 | glm-4v-flash |
| 相似度阈值 | 超过此值标记可疑 | 80% |
| 预筛选阈值 | pHash 汉明距离上限 | 10 |

配置存储在 localStorage，首次使用弹出配置引导。

---

## 数据存储方案

| 数据 | 存储位置 | 说明 |
|------|---------|------|
| API Key/配置 | localStorage | 文本数据，体积小 |
| 图片库 | IndexedDB | 图片 base64 体积大，IndexedDB 无容量限制 |
| 本次上传图片 | 内存 | 仅当次会话有效 |
| 分析结果 | 内存 | 刷新即丢失 |

---

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| API Key 未配置 | 弹出配置引导，阻止分析 |
| API 调用超时或失败 | 单张失败不影响其他，标记为"分析失败，可重试" |
| 图片无法识别类型 | 标记为"未知类型"，跳过对比，报告中列出 |
| 库中无基准图 | 提示用户先上传基准图到图片库 |
| 上传非支持格式 | 前端校验，拒绝上传并提示 |
| API 余额不足 | 提示用户更换 Key |
