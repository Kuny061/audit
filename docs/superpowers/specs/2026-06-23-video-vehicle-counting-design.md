# 视频航拍车辆统计 — 设计文档

## 概述

在现有 AI 智能审计分析平台中新增视频上传和航拍车辆统计功能。用户上传航拍视频（渣土车等），通过截帧工具截取画面，调用 GLM Vision API 统计截图中车辆数量及位置。

与现有审计流程完全解耦，不复用已有模态框。

## 入口

复用现有上传区（`#aiUploadArea`）。`handleFileSelect()` 中检测视频文件（`file.type.startsWith('video/')`），自动弹出视频分析模态框。图片文件行为不变。

## 视频分析模态框

### 结构

独立模态框 `#aiVideoModal`，三层视图切换：

1. **截帧视图** — 视频播放器 + 截图工具 + 多帧缩略图列表
2. **分析中视图** — loading 状态
3. **结果视图** — 截图 + 车辆统计 + 位置描述

### 截帧视图

```
┌──────────────────────────────────────────────┐
│ 视频截图分析 — 渣土车航拍统计                    │
├────────────────────────────────┬─────────────┤
│                                │ 已截帧 N 张   │
│     视频播放区域               │ ┌─────────┐  │
│     (video element)           │ │ 缩略图1  │  │
│                                │ │ 00:15 ✕ │  │
│                                │ ├─────────┤  │
│                                │ │ 缩略图2  │  │
│                                │ │ 01:22 ✕ │  │
│                                │ └─────────┘  │
│  ⏪-5s ▶ 播放 ⏩+5s  00:42/03:15│              │
│  ───●──────────── timeline     │ [清空重截]    │
│                                │ [📤上传分析]  │
│      [📷 截取当前帧]            │              │
└────────────────────────────────┴─────────────┘
```

- 视频播放控制：播放/暂停、±5秒跳转、时间轴滑块、时间显示
- 截帧按钮：canvas.drawImage(video) → toBlob → base64 → 存入 frames[]
- 右侧缩略图列表：每张显示截帧时间点，悬停显示删除按钮
- 底部按钮："清空重截"（删除所有帧）、"上传分析"（进入分析）

### 分析中视图

居中 loading 动画 + "正在调用 AI 模型分析截帧中的车辆..." 文案。

每帧独立调用 API，逐帧完成。单帧失败不影响其他帧。

- 顶部标签页切换各帧结果，右侧合计总数
- 左侧显示截帧图片
- 右侧显示车辆总数（大字）和位置描述列表
- 底部提供"重新分析"按钮（返回截帧视图可重新截图）

## API 调用

### 复用现有配置

使用 `AuditAPI.loadConfig()` 读取 apiKey、apiBase、model。复用 `AuditAPI._callAPI()`。

### Prompt

```
这是一张航拍图片。请统计图片中渣土车（自卸货车）的数量，并描述每辆车在图片中的大致位置。

请仅返回JSON（不要其他内容）：
{"count": 数字, "vehicles": [{"id": 序号, "position": "位置描述", "color": "颜色", "status": "行驶中/停靠"}]}
```

### 并行策略

逐帧串行调用，每帧完成后更新对应结果。简单可靠，避免并发 API 限流。

## 数据管理

| 数据 | 存储 | 生命周期 |
|------|------|---------|
| 视频文件 | 内存 (blob URL) | 模态框关闭时 revoke |
| 截帧图片 | 内存 (base64) | 模态框关闭时释放 |
| 分析结果 | 内存 | 模态框关闭时释放 |
| API 配置 | localStorage | 持久 |

不写入 IndexedDB，不入图片库。

## 错误处理

| 场景 | 处理 |
|------|------|
| 视频格式不支持 | `video.onerror` → 提示"不支持该视频格式，请使用 MP4/MOV 等常见格式" |
| API Key 未配置 | 阻止分析，提示"请先配置 API Key"，展开配置面板 |
| 单帧分析失败 | 标记 `frame.error = msg`，显示"分析失败"标签，继续分析其他帧 |
| 截帧数为0时点上传 | 按钮置灰（`disabled`），不可点击 |
| 视频加载失败 | 提示"视频加载失败，请检查文件" |

## 与现有功能的关系

```
handleFileSelect(files)
  ├── file.type.startsWith('image/') → 现有图片上传流程（不变）
  └── file.type.startsWith('video/') → VideoAnalysisModal.open(file)
```

两条路径完全独立，互不干扰。视频分析不经过三阶段管线、不入库、不生成审计报告。
